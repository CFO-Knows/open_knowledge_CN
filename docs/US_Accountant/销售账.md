# 餐饮公司进出账

进账：

* 销售收入

出账：

* 日常开销
* 员工工资
* 购买货物
* 租金房屋支出等

那为什么要从销售中知道detail？老板想要分析sales。从真实的sales中分析明天后天要怎么去梳理账。另一方面则是关于销售税。

餐厅只在一个点销售，所以销售上会清晰一点。小系统就搞不了了。卖到不同的city和不同的state会导致sales tax不一样。

1万块钱的现金收入：

* sales收入是10000块，记成一笔（银行账是第一可靠的点）
* 对总账来说没有差别，但是detail在audit的时候是需要的。（audit频繁吗？不太频繁，所以餐厅的记录可以先不上传，自己保留好就行。audit一般三五年后才会来，一般都会被失败，所以一般都要罚钱付款）

财务报表的产生不需要记的很细，比如10000是怎么分的。

会计原则是发生sales就算，不一定是代收款。中小企业approve basis会计原则上会宽松一点，发生就算。

必须要考虑的点：

* adjustment功能，比如调整
* 以及系统能够识别出不match的地方，好调整的话就先自动调整，不好调整的话就询问（弹出一个自然语言沟通界面，比如问这一笔现金具体是什么情况）
* 不在银行的东西，开出的票之类的，有没有，需要单独记的（比如5张invoice，还没收钱）
* 要有开invoice的功能，自动寄送到email或者打印出来之类的；由于没有国家具体的要求，所以可以设置一些模板，用户可以自己选。
